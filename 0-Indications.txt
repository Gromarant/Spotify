Ejercicios
1. Crea el proyecto
    1. Crea un repositorio nuevo en GitHub llamado promo-X-module-1-pair-Y-spotify. Hazlo en la organización de Adalab. (Si aún no lo tenemos claro debemos revisar la guia básica de git, o esperar a la clase de git, no es prioridad este punto)
    2. En este enlace se encuentra el diseño para las diferentes versiones de dispositivos.(https://github.com/Gromarant/ejercicios-de-los-materiales/tree/main/html-spotify)
    3. Crear la estructura de carpetas correcta para tu proyecto.HTML, CSS

2. Crea todo el contenido y estructura de tu HTML
    1. Crear todo el contenido y etiquetas del HTML, prestando especial atención a la carga semántica que debemos implementar en cada sección de nuestra maqueta.
    2. Una vez hemos creado nuestro contenido es hora de comenzar a darle estilo, y para ello vamos a comenzar creando una hoja de estilo de reset para eliminar las características por defecto que nos convenga.
    
3. Pasos para crear el código CSS
    Debemos empezar a maquetar desde los elementos más exteriores hacia los más interiores. Esto es porque los estilos CSS de los elementos externos pueden ser heredados por los elementos más internos, y por lo tanto modificarlos.

    1. Tenemos que identificar qué grupos de elementos se comportan como un único elemento. Sabremos identificar si están visualmente juntos y si se comportan de la misma manera en diseño móvil, tablet y desktop. Por ejemplo, si varios elementos se centran todos a la vez al pasar de móvil a desktop nos dan una pista de que puede ser un grupo de elementos. Para esto nos viene muy bien coger rotu y papel y pintar la página.

    2. Una vez identificados estos grupos de elementos que se comportan como uno solo tenemos que agruparlos en el código. Para ello los envolvemos en un contenedor. Si ya estaban agrupados en un contenedor con carga semántica por ejemplo <header />, no hace falta que los volvamos a agrupar.

    3. A continuación le tenemos que añadir una clase css a cada uno de estos contenedores. Y vamos a dar los siguientes estilos a cada clase: .my-container { border: solid 3px red; background: blue} (debemos sustituir my-container por el nombre de la clase que le hayamos dado). Con esto conseguiremos identificar visualmente los contenedores. Es decir es simplemente para saber el espacio que ocupa cada contenedor en nuestra maqueta. Una vez hayamos terminado de maquetar todos los contenedores, podremos borrar los estilos de borde y fondo que habíamos puesto para visualizar fácilmente los contenedores. Y podremos empezar a maquetar fino fino y al detalle cada elemento de la página, centrándonos en detalles como colores, tamaños de fuente, márgenes, etc...

    4. Por último nos vamos a centrar en la cabecera y comenzar a aplicar estilos que correspondan a la imagen y al menú indicado en el diseño. El fondo de la cabecera siempre va a ser negro. La letra blanca.
